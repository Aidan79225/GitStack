GitCrisp

A clean, focused Git GUI

Visual commit graph, per-hunk staging, interactive rebase, conflict resolution, and full management dialogs for branches, remotes, submodules, and tags — built with Python and Qt.

Download View on GitHub

Features

📊

Visual Commit Graph

Lane-based graph with lazy pagination, inline search (Ctrl+F), branch navigation, and click-to-copy commit hash. Auto-refresh on focus return preserves your scroll position.

🧩

Per-Hunk Staging

Stage or unstage individual diff hunks. Lazy diff loading with skeleton placeholders for smooth handling of large commits. Collapsible commit message and per-file diff blocks let you skip past noise.

🌿

Branch & Rebase

Merge options dialog (ff-only / no-ff / allow-ff), interactive rebase with pick / squash / fixup / drop, and a checkout-conflict reset prompt.

⚠️

Conflict Resolution

Abort / Continue banner in both panels, conflict file indicators, per-block conflict hunks, and dual-parent graph during merges.

🏷️

Tags & Stash

Create, delete, and push tags. A Stash button appears in the commit graph header when you have uncommitted changes — confirms before stashing. View contents inline, pop / apply / drop from context menu.

🌐

Remote Operations

Push, pull, fetch with status bar indicator. Force push prompt on rejection (confirmation message box offers --force-with-lease). Delete remote branches from the sidebar OR the graph context menu. Remotes CRUD dialog. Background threading.

🔗

Submodule Support

Manage submodules from a dedicated dialog, click diffs to jump into submodule repos, and detect submodule HEAD divergence in the parent.

📂

Multi-Repository

Two-line repo entries for disambiguation, drag-and-drop reordering, submodule grouping, and keyboard shortcuts (Ctrl+W, Ctrl+1-9).

🎨

Theming & Insights

Softer primary-tinted Light theme and a Material 3 Dark, plus a global typography scale that persists across restarts. Custom panel doubles as an inspection tool — open it in any mode to read every token's hex code. Per-author commit stats stream incrementally with progress and cancellation, so even All on a long-lived repo is workable.
